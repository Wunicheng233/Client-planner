# 智能移动技术大作业 - 仿真路径规划与控制

这是浙江大学控制学院《智能移动技术》大作业的开源仓库！

本项目基于 RoboCup SSL（小型组足球机器人）仿真平台，实现了一套高鲁棒性、极速的**全局路径规划**与**局部避障控制系统**。通过与本地 Client 仿真环境进行 UDP 通信，控制蓝车 0 号在复杂的动态/静态环境中完成往返跑任务。

本仓库代码已在验收中取得了优异的成绩，可供大家参考！

## 🌟 性能概览

根据课程要求（起点 `(2500, 1800)` ↔ 终点 `(-2400, -1500)`，5 个来回），本系统在最高限速（线速度 5000 mm/s，角速度 10 rad/s）下达到了以下性能：

* **速度合理**：在 SIMPLE 与 HARD 模式下，5 个来回的总耗时均稳定压缩在 **90秒以内**（满分线为 <100s）。
* **零碰撞/极低碰撞**：面对高速无规则运动的动态黄车，碰撞次数稳定控制在 **0~2 次**。
* **轨迹丝滑**：轨迹平滑优美，无原地打转、无频繁急停急转、无“闭眼冲锋”导致的穿模死锁。

## 🧠 核心算法架构（我们的通关秘籍）

本项目没有堆砌臃肿的算法（我们在开发期曾尝试过 DWA 但因死锁问题果断弃用），而是秉持**“大道至简、工程鲁棒”**的原则，采用了一套极其高效的组合拳：

1.  **感知层 - 时空残影墙 (Ghost Obstacles)**：不局限于“眼见为实”。利用视觉数据提取动态黄车的速度矢量，提前在其前方 0.2~0.6 秒处生成连续的虚拟残影，赋予系统“预知未来”的动态避让能力。
2.  **规划层 - 安全代价梯度 A* (Cost Map A*)**：传统的 A* 容易“贴墙走”导致过弯剐蹭。我们在 A* 中引入了软边界代价梯度，迫使规划出的路径主动向宽敞区域偏移，为高速漂移留足物理余量。
3.  **平滑层 - 滑动均值滤波 (Moving Average Filter)**：摒弃了会丢失节点的视线剪枝法。使用滑动均值滤波像熨斗一样烫平 A* 的网格锯齿，且**不减少任何一个路径节点**，完美保留了局部雷达的密集扫描防撞能力。
4.  **控制层 - 非线性前瞻 PD 追踪 (Non-linear PD)**：采用前瞻点纯追踪逻辑（Lookahead=250mm）。创新性引入**“慢进快出”的平方级重刹车机制**：车头偏角稍大即断崖式降速，出弯对正后瞬间全油门拉满，极致压榨底盘性能。

## 📁 项目结构说明

* `main.py`：主程序入口，仅负责模块的实例化和主循环启动。极度清爽，不到 20 行。
* `utils.py`：**核心主战场**。包含了最核心的 `RobotController` 类（状态机管理、动态残影生成、PD 控制律、滤波平滑等）。
* `a_star_planner.py`：全局路径规划器。内置了带有安全代价梯度的定制版 A* 算法。
* `vision.py` / `action.py` / `debug.py`：底层 UDP 通信与接口封装，保持原样即可。
* `*_pb2.py`：Protobuf 自动生成的数据结构文件，绝对不要修改。

## 🛠️ 环境配置指南

为避免底层依赖（特别是 Protobuf）的玄学报错，**强烈建议使用 Conda 隔离环境**：

**1. 克隆代码仓库**
```bash
git clone https://github.com/Wunicheng233/Client-planner.git
cd Client-planner
```

**2. 创建并激活 Conda 环境 (推荐 Python 3.8)**
```bash
conda create -n move python=3.8 -y
conda activate move
```

**3. 安装核心依赖 (版本极其重要！)**
本项目依赖 `protobuf` 进行数据序列化。**注意：版本过高会导致运行时直接报错退出！** 必须严格指定安装 `3.20.0` 版本：
```bash
pip install protobuf==3.20.0
```

## 🚀 运行与测试

1.  **启动仿真器**：打开桌面的 `Client` 软件。
2.  **连接图像**：点击右侧的“连接图像”按钮（插头图标），选择测试场景（`SIMPLE` 或 `HARD`）。
3.  **运行主程序**：在激活了 `move` 环境的终端中，运行：
    ```bash
    python main.py
    ```

> **成功标志**：终端打印 `>>> A* + PD 追踪器就绪 <<<`，Client 中的蓝车 0 号开始走位，并在界面上画出绿色规划轨迹与白色目标前瞻点。
> *(Tip: 在 Client 中按住 `Ctrl` 键加上数字键可以快速召唤多辆黄车，鼠标也可直接拖拽小车测试避障响应)*

## 💡 常见错误排查 

| 报错现象 / 提示语 | 产生原因 | 解决办法 |
| :--- | :--- | :--- |
| **Client 运行白屏** | OpenGL 渲染库缺失 | 在系统 `C:\Windows\System32` 路径下找到 `opengl32.dll`，将其复制并覆盖粘贴到 Client 的安装目录下。 |
| **`TypeError: Descriptors cannot not be created directly`** | `protobuf` 版本过高 | 终端执行 `pip install protobuf==3.20.0` 强制降级。 |
| **`OSError: [WinError 10048] 通常每个套接字地址只允许使用一次`** | UDP 端口被占用 | 确保后台没有没关掉的 Python 进程。在 Client 中点击“快速关闭程序”（靶心图标）彻底关闭软件后，重新打开。 |
| **终端无报错，但小车不动 / 视觉坐标全是 -999999** | 视觉模块未成功建立连接 | 在 Client 中断开图像连接后，重新点击连接。 |

## 🎓 给学弟学妹的忠告

1.  **别盲目迷信高级算法**：在这门课的仿真环境下（高延时、离散帧率），复杂的 DWA 或 APF 极易陷入局部死锁。调好一个带代价地图的 A* 加上扎实的 PD 控制，比什么花里胡哨的算法都管用。
2.  **敬畏物理惯性**：代码里的路算得再好，车子 5000 速度冲进弯道也一定会往外飘。永远记得在规划层给障碍物多留几厘米的“膨胀半径”。
3.  **防抖机制是高分的细节**：仿真器的判定很苛刻，边缘剐蹭会导致计分器在一秒内狂跳。记得在代码里加上碰撞冷却时间（Debounce），保护你的平时分。
